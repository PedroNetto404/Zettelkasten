# Orientação a objetos - Livro
A diferença entre programação procedural e programação orientada a objetos é que na primeira nos preocupamos principalmente com a implementação e a escolha do melhor algoritmo para resolver determinado problema e na segunda, por mais que nos preocupemos também com a implementação, a ideia principal é focarmos no projeto de classes, como elas se comunicam e se encaixam no todo, pensamos o software como se fosse um quebra cabeça onde cada parte tem sua importância e forma de se encaixar com outras; 

A coessão de código é atingida quando modelamos nossas de uma forma que cada entidade cuide de um e apenas 1 conceito no sistema, que maneira que poderemos reaproveita-la em difentes partes do código. 
Uma classe possui dentro de sí toda uma estrutura de dados e diversas funcionalidades que dizem respeito principamente ao comportamento de processamento de dados internos (encapsulamento); 
Para atingirmos uma melhor coesão devemos pensar criticamente a respeito de classes que tendem a crescer indefinidademente. Essa situação representa que essa classe pode estar acumulando muitas tarefas,  o que nos diz para quebra-la em classes menores para uma melhor coesão de código; 

Métodos privados devem ser utilizados para melhorar a legibildiade de métodos públicos. Note que métodos privados podem ser acessados apenas pela própria classe, isto é, não podem ser acessamos por classes clientes. Logo, não faz sentido darmos responsabilidades de funcionalidades chave para métodos privados encapsulando de forma errônea comportamentos que poderiam ser reaproveitados no furuto por outro ponto do sistema.
